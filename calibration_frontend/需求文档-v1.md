# 前端开发需求
## 一、基础信息
1. 开发类型：【完整页面】
2. 技术栈要求：【Vue3+Vite+Element Plus】
3. 适配要求：【仅PC端（分辨率：1920*1080）】
4. 交付物：【代码+简单说明文档】
5. 系统名称：手眼标定平台
6. 用户群体：蓝领工人通过本系统自动实现手眼标定

## 二、功能需求
### 2.1 核心功能
【1. 用户可以观看一个完整的手眼标定的例子，并学习该流程；2. 引导用户实现手眼标定；3. 引导用户验证手眼标定的结果】

### 2.2 交互逻辑
【先给出一个大概的骨架，后续我继续完善】

### 2.3 特殊交互/动画
【无】

## 三、UI/样式要求
1. 配色方案：【主色#1890ff，辅助色#f5222d，中性色#f5f5f5（背景）、#333333（文字）】
2. 字体要求：【全局字体微软雅黑/Inter，标题字号18px（加粗），正文14px，辅助文字12px】
3. 布局要求：【页面上下布局，顶部导航栏高度60px，主体内容区左右布局，左侧侧边栏宽度200px，右侧内容区自适应】
4. 参考样式：【参考Element Plus官方表格样式/参考某网站（链接）的按钮样式】

## 四、数据相关
1. 数据来源：【模拟假数据/接口请求（接口地址：xxx，请求方式：GET/POST）/本地localStorage】
2. 数据格式：【用户列表格式：[{id: 1, name: '张三', phone: '13800138000'}, ...]】
3. 数据处理：【手机号中间4位脱敏/时间戳转换为YYYY-MM-DD格式/列表数据按创建时间倒序排列】

## 五、其他要求
1. 兼容性：【兼容主流浏览器】
2. 性能要求：【列表数据超过100条时使用虚拟滚动/首次加载时间不超过2s】
3. 代码规范：【使用ES6语法/组件化开发/添加必要的注释/遵循Eslint规范】
4. 额外说明：【需要封装通用的请求函数/代码需具备可复用性/需处理接口请求失败的异常提示】
